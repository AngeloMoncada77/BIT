HTML semántico: la base del SEO y la accesibilidad

Julio 2026 · Programación

Escribe HTML como si Google y una persona con discapacidad visual fueran a leerlo. Porque lo harán. El HTML semántico no es una moda ni una buena práctica abstracta: es lo que determina si tu sitio aparece en los resultados de búsqueda y si es utilizable por todos.

¿Qué es HTML semántico?

HTML semántico significa usar etiquetas que describen el significado del contenido, no solo su apariencia. En lugar de llenar todo con <div> y <span>, usas etiquetas como <header>, <nav>, <main>, <article>, <section>, <aside> y <footer>.

Compara estos dos ejemplos:

<!-- No semántico -->
<div class="header">
  <div class="nav">...</div>
</div>
<div class="main">
  <div class="article">...</div>
</div>

<!-- Semántico -->
<header>
  <nav>...</nav>
</header>
<main>
  <article>...</article>
</main>

Ambos se ven idénticos visualmente. Pero el segundo le dice al navegador, al lector de pantalla y al motor de búsqueda exactamente qué es cada cosa.

Beneficio 1: SEO

Google usa la estructura HTML para entender tu contenido. Cuando usas <h1> para el título principal, <h2> para subtítulos, y <article> para contenido independiente, estás ayudando al algoritmo a indexar tu página correctamente.

Los rich snippets (resultados enriquecidos que aparecen en Google) dependen de datos estructurados, pero una base de HTML semántico es el primer paso para que Google entienda la jerarquía de tu página.

Reglas de oro:

  • Un solo <h1> por página
  • Los encabezados siguen jerarquía: h1 → h2 → h3 (no saltos)
  • Cada <article> puede tener su propia jerarquía de encabezados
  • <nav> debe contener solo enlaces de navegación principales

Beneficio 2: Accesibilidad

Las personas que usan lectores de pantalla navegan por puntos de referencia (landmarks). Un <header> es un landmark. Un <div class="header"> no lo es. La diferencia es invisible en pantalla, pero crítica para quien usa un screen reader.

Además, HTML semántico ayuda con:

  • Navegación por teclado: los elementos interactivos semánticos (<button>, <a>) son accesibles por Tab por defecto
  • Textos alternativos: <img alt="descripción"> es obligatorio para lectores de pantalla
  • Formularios: usar <label for="id"> en lugar de solo <input placeholder>

Beneficio 3: Mantenibilidad

Cuando vuelves a tu código 6 meses después, <article> te dice inmediatamente que ese bloque es contenido independiente. <aside> te dice que es contenido relacionado pero no central. No necesitas leer todo el CSS para entender la estructura.

Las etiquetas semánticas esenciales

  • <header> — Introducción o navegación de una sección
  • <nav> — Bloque de navegación principal
  • <main> — Contenido principal (único por página)
  • <article> — Contenido independiente y reutilizable
  • <section> — Agrupación temática de contenido
  • <aside> — Contenido complementario (sidebar)
  • <footer> — Pie de página de una sección o del documento
  • <figure> / <figcaption> — Imagen o ilustración con leyenda
  • <time> — Fechas y horas (legible por máquinas)

El HTML semántico no cuesta nada, no ralentiza tu página y no requiere librerías externas. Es simplemente elegir la etiqueta correcta para el trabajo correcto. Hazlo desde el día uno y tu yo del futuro (y Google) te lo agradecerán.
