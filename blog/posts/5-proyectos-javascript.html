5 proyectos con JavaScript puro para tu portafolio en 2026

Julio 2026 · Programación

En 2026, los reclutadores han visto demasiados "TODO apps" y "clones de Netflix". Para destacar, necesitas proyectos que resuelvan problemas reales y demuestren comprensión profunda de JavaScript, no solo capacidad de seguir tutoriales.

Aquí tienes 5 proyectos con JS vanilla (sin frameworks) que realmente llaman la atención:

1. Flashcards Generator

Una herramienta que toma texto y genera automáticamente tarjetas de estudio. El usuario pega sus apuntes y la app identifica conceptos clave para crear pares de pregunta-respuesta.

Habilidades que demuestra: manipulación del DOM, procesamiento de texto, lógica de estado (navegación entre tarjetas), persistencia de sesión, diseño UX (feedback visual al hacer clic).

Extra: añade modo de arrastrar y soltar para organizar las tarjetas por prioridad.

2. Grade Calculator

Una calculadora que responde a la pregunta "¿qué nota necesito en el final para obtener X?" El estudiante ingresa su nota actual, la nota deseada y el peso del examen, y la app calcula el resultado.

Habilidades que demuestra: manejo de formularios, validación de entrada, lógica matemática, renderizado condicional (resultados positivos vs negativos con colores distintos).

3. Dashboard con datos de APIs públicas

Construye un tablero que muestre datos en vivo de una o más APIs públicas. Ejemplos: clima, precios de criptomonedas, actividades de GitHub, estado del mercado de valores.

Habilidades que demuestra: fetch asíncrono, manejo de promesas y errores, renderizado de datos dinámicos, diseño de grids responsivos, auto-refresh periódico.

async function fetchData(url) {
  try {
    const res = await fetch(url);
    if (!res.ok) throw new Error('Error ' + res.status);
    return await res.json();
  } catch (err) {
    showError(err.message);
  }
}

4. Pomodoro Timer con notificaciones

Un temporizador Pomodoro completo: 25 minutos de trabajo, 5 de descanso, con ciclo automático. Incluye notificaciones visuales y sonoras al cambiar de fase, y contador de ciclos completados.

Habilidades que demuestra: setInterval/setTimeout, manejo de estado de juego (corriendo, pausado, reinicio), eventos de teclado (espacio para pausar/reanudar), Notification API del navegador.

5. Markdown Editor en vivo

Un editor dividido en dos paneles: a la izquierda se escribe Markdown, a la derecha se ve el HTML renderizado en tiempo real. Incluye exportación a HTML y copia al portapapeles.

Habilidades que demuestra: parseo de texto, renderizado en tiempo real (input → output), separación de responsabilidades, uso de contentEditable, Clipboard API.

Puedes escribir tu propio parser Markdown simple (encabezados con #, listas con -, negritas con **) o usar una librería como marked.js.

Cómo presentar estos proyectos

No basta con construirlos. La presentación importa tanto como el código:

  • Demo en vivo: desplegado en Netlify, Vercel o GitHub Pages
  • README sólido: qué problema resuelve, stack, cómo ejecutarlo localmente
  • Código limpio: funciones pequeñas, nombres descriptivos, sin comentarios redundantes
  • Captura o GIF: muestra la app funcionando en el README

Construye 3 de estos proyectos, preséntalos bien, y tendrás un portafolio que dice mucho más que cualquier certificado.
