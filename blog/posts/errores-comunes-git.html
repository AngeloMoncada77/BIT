Errores comunes de Git y cómo solucionarlos

Git es una herramienta increíble, pero sus mensajes de error pueden ser crípticos. Aquí tienes los errores más frecuentes y cómo resolverlos paso a paso.

1. "failed to push some refs to"

Este error ocurre cuando tu repositorio local está detrás del remoto. La solución es traer los cambios del remoto primero:

git pull origin main --rebase
git push origin main

Si usas main o master, ajusta según el nombre de tu rama principal.

2. Merge conflicts

Ocurren cuando dos personas modifican la misma parte de un archivo. Git marca las secciones conflictivas así:

<<<<<<< HEAD
tu código
=======
código del otro
>>>>>>> nombre-rama

Para resolverlo:

  • Edita el archivo y decide qué versión conservar
  • Elimina los marcadores <<<<<<<, ======= y >>>>>>>
  • Guarda el archivo y ejecuta:
git add archivo-resuelto
git commit

3. "detached HEAD"

Significa que no estás en ninguna rama, sino en un commit específico. Para solucionarlo:

git branch nueva-rama
git checkout nueva-rama

O si quieres volver a una rama existente sin conservar cambios:

git checkout main

4. "fatal: not a git repository"

Estás en una carpeta que no tiene un repositorio Git. Inicialízalo con:

git init

O muévete a la carpeta correcta que contiene la carpeta .git.

5. "Changes not staged for commit"

Modificaste archivos pero no los agregaste al área de staging. Solución:

git add .           # Agrega todo
git add archivo.js  # Agrega un archivo específico
git commit -m "mensaje"

6. "fatal: remote origin already exists"

Ya existe un remoto configurado. Puedes actualizarlo en lugar de crearlo:

git remote set-url origin https://github.com/usuario/repo.git

7. Cómo deshacer un commit

Si hiciste un commit y quieres deshacerlo:

git reset --soft HEAD~1   # Quita el commit pero mantiene los cambios
git reset --hard HEAD~1   # Quita el commit y los cambios (¡cuidado!)

8. "Your branch is ahead of origin/main"

Tienes commits locales que no están en el remoto. Solo haz push:

git push origin main

Consejo final

Si alguna vez te sientes perdido, git status y git log --oneline son tus mejores amigos. Muestran exactamente en qué estado está tu repositorio.
