CSS Grid: La guía completa para maquetar con cuadrícula

Si alguna vez has tenido que maquetar una página completa con header, sidebar y footer, sabes que Flexbox se queda corto. CSS Grid Layout llegó para revolucionar la maquetación web, permitiéndonos crear layouts en dos dimensiones (filas y columnas) de forma sencilla y predecible. En esta guía veremos desde los conceptos básicos hasta ejemplos prácticos que puedes usar hoy mismo.

¿Qué es CSS Grid?

CSS Grid es un sistema de maquetación bidimensional nativo del navegador. A diferencia de Flexbox, que trabaja en una sola dimensión (horizontal o vertical), Grid te permite controlar filas y columnas al mismo tiempo. Piensa en una hoja de cálculo: puedes colocar elementos en celdas específicas, hacer que ocupen varias celdas o incluso superponerlos.

¿Por qué es mejor que Flexbox para layouts 2D?

Flexbox es excelente para distribuir elementos en una línea (como una barra de navegación o una fila de tarjetas), pero cuando necesitas alinear elementos en ambas direcciones simultáneamente, Grid es la herramienta correcta. Con Grid defines la estructura general de la página mientras que Flexbox se encarga de los detalles dentro de cada celda. Se complementan, no compiten.

Conceptos clave

Antes de escribir código, familiaricémonos con la terminología de CSS Grid:

  • Grid Container: El elemento padre al que se le aplica display: grid. Es el que define la cuadrícula.
  • Grid Items: Los hijos directos del contenedor. Se colocan automáticamente en las celdas de la cuadrícula.
  • Grid Lines: Las líneas divisorias entre filas y columnas. Se numeran empezando desde 1.
  • Grid Tracks: El espacio entre dos líneas consecutivas, es decir, una fila o una columna.
  • Grid Cells: La intersección de una fila y una columna. Es la unidad más pequeña de la cuadrícula.
  • Grid Areas: Una región rectangular formada por una o más celdas. Se definen con grid-template-areas.

Propiedades del contenedor

Estas propiedades se aplican al elemento que funciona como grid container:

display: grid

Convierte un elemento en un contenedor de cuadrícula. Sus hijos pasan a ser grid items.

grid-template-columns y grid-template-rows

Definen el tamaño y la cantidad de columnas y filas. Puedes usar cualquier unidad: px, %, fr, auto, minmax(), etc.

.grid {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  grid-template-rows: auto 1fr auto;
}

La unidad fr reparte el espacio disponible de forma proporcional. En este ejemplo, la columna central ocupará el doble que las laterales.

gap

Define el espacio entre filas y columnas. Es la forma moderna de separar elementos sin necesidad de márgenes.

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}

grid-template-areas

Te permite asignar nombres a las áreas de tu cuadrícula para colocar los items de forma visual.

.grid {
  display: grid;
  grid-template-columns: 1fr 3fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
}

justify-items y align-items

Controlan la alineación de los items dentro de sus celdas en el eje horizontal y vertical respectivamente. Aceptan valores como start, end, center y stretch (valor por defecto).

Propiedades de los items

Se aplican a los hijos directos del grid container (los grid items):

grid-column y grid-row

Permiten que un item ocupe múltiples columnas o filas especificando las líneas de inicio y fin.

.item {
  grid-column: 1 / 3; /* de línea 1 a línea 3 */
  grid-row: 1 / 2;
}

/* También puedes usar "span" */
.item {
  grid-column: span 2; /* ocupa 2 columnas */
}

grid-area

Asigna un item a un área nombrada definida con grid-template-areas, o como atajo de grid-row-start / grid-column-start / grid-row-end / grid-column-end.

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.footer { grid-area: footer; }

justify-self y align-self

Sobrescriben justify-items y align-items para un item concreto.

Ejemplo 1: Grid básico de 3 columnas responsivo

Usando auto-fit y minmax() conseguimos un grid que se adapta automáticamente al ancho disponible sin media queries:

.grid-auto {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1.5rem;
}

Cada tarjeta tendrá un mínimo de 250 px y, si hay espacio, se estirará. Cuando no quepan más en una fila, pasan a la siguiente. Es ideal para galerías o listas de productos.

HTML de ejemplo

<div class="grid-auto">
  <div class="card">Tarjeta 1</div>
  <div class="card">Tarjeta 2</div>
  <div class="card">Tarjeta 3</div>
  <div class="card">Tarjeta 4</div>
  <div class="card">Tarjeta 5</div>
</div>

Ejemplo 2: Layout de página completo

Este es el uso más potente de Grid: definir la estructura completa de una página usando grid-template-areas:

.page-layout {
  display: grid;
  grid-template-columns: 250px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  min-height: 100vh;
  gap: 0;
}

.page-header  { grid-area: header;  background: #1e293b; color: #fff; padding: 1rem 2rem; }
.page-sidebar { grid-area: sidebar; background: #f1f5f9; padding: 1.5rem; }
.page-main    { grid-area: main;    padding: 1.5rem; }
.page-footer  { grid-area: footer;  background: #1e293b; color: #fff; padding: 1rem 2rem; text-align: center; }

Con este enfoque, el orden de los elementos en el HTML no importa. El navegador los coloca según el área asignada. Además, para hacerlo responsivo solo necesitas redefinir grid-template-areas:

@media (max-width: 768px) {
  .page-layout {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "sidebar"
      "main"
      "footer";
  }
}

Ejemplo 3: Galería de imágenes con grid automático

Combinando grid-auto-flow: dense con span podemos crear una galería tipo masonry sin JavaScript:

.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  grid-auto-flow: dense;
  gap: 0.5rem;
}

.gallery img {
  width: 100%;
  height: 100%;
  object-fit: cover;
  display: block;
}

.gallery img.wide { grid-column: span 2; }
.gallery img.tall { grid-row: span 2; }
.gallery img.big  { grid-column: span 2; grid-row: span 2; }

La propiedad grid-auto-flow: dense hace que los items ocupen los huecos que dejan los elementos más grandes, optimizando el espacio disponible.

Diferencia con Flexbox: ¿cuándo usar cada uno?

Una de las dudas más comunes entre quienes empiezan es saber cuándo usar Grid y cuándo Flexbox. Aquí tienes una guía rápida:

  • Usa Flexbox cuando necesites distribuir elementos en una sola dirección: una barra de navegación, una fila de botones, centrar un elemento vertical y horizontalmente.
  • Usa Grid cuando necesites controlar filas y columnas al mismo tiempo: el layout general de una página, una galería de imágenes, un dashboard con paneles.
  • Usa ambos en el mismo proyecto. Grid para el layout macro de la página y Flexbox para los componentes dentro de cada celda del grid.

No hay una regla estricta. Si tu layout se siente forzado con uno, prueba con el otro. La práctica te dará el criterio.

Conclusión

CSS Grid ha cambiado la forma en que maquetamos para la web. Con unas pocas líneas de CSS puedes crear layouts complejos que antes requerían floats, clearfixes y frameworks enteros. Lo mejor de todo es que Grid es nativo del navegador, no necesitas librerías externas, y funciona en todos los navegadores modernos.

Empieza usándolo en proyectos pequeños: una galería, un layout de blog, un dashboard. Una vez que te acostumbres a pensar en términos de filas y columnas, no querrás volver atrás.
