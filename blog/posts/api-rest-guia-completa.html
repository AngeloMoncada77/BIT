¿Qué es una API REST? Guía completa para entender y consumir APIs

Si has desarrollado alguna aplicación web moderna, seguro has escuchado el término "API REST". Pero, ¿qué significa exactamente? En este artículo vamos a desglosar qué es una API, qué hace que sea REST, cómo funcionan los métodos HTTP y cómo puedes consumir APIs desde JavaScript y Python. Al final, crearás una petición real a una API pública.

¿Qué es una API?

API son las siglas de Application Programming Interface (Interfaz de Programación de Aplicaciones). En términos sencillos, una API es un intermediario que permite que dos aplicaciones se comuniquen entre sí.

Imagina un restaurante: tú eres la aplicación cliente, la cocina es el servidor, y el mesero es la API. Tú no entras a la cocina a preparar tu comida; le dices al mesero lo que quieres y él te trae el resultado. La API funciona igual: recibe tu solicitud, se la comunica al servidor y te devuelve la respuesta.

¿Qué significa REST?

REST (Representational State Transfer) es un conjunto de principios arquitectónicos para diseñar APIs. No es un protocolo ni un estándar, sino una guía de buenas prácticas. Una API que sigue estos principios se denomina RESTful.

Los principios clave de REST son:

  • Cliente-servidor: El cliente y el servidor son independientes. El cliente solo conoce la URL del recurso que necesita.
  • Sin estado (stateless): Cada petición del cliente contiene toda la información necesaria para ser procesada. El servidor no guarda el estado de la sesión.
  • Cacheable: Las respuestas deben indicar si pueden ser cacheadas o no, para mejorar el rendimiento.
  • Interfaz uniforme: Los recursos se identifican mediante URLs y se manipulan con métodos HTTP estándar.

Métodos HTTP

En una API REST, las acciones se representan mediante métodos HTTP. Los más comunes son:

  • GET: Obtener un recurso o lista de recursos. Es el método más usado.
  • POST: Crear un nuevo recurso.
  • PUT: Actualizar un recurso existente por completo.
  • PATCH: Actualizar parcialmente un recurso.
  • DELETE: Eliminar un recurso.

Ejemplo de URLs RESTful

Supongamos una API de usuarios. Las URLs seguirían esta estructura:

GET    /usuarios          → Obtener todos los usuarios
GET    /usuarios/42       → Obtener el usuario con ID 42
POST   /usuarios          → Crear un nuevo usuario
PUT    /usuarios/42       → Actualizar el usuario 42
PATCH  /usuarios/42       → Actualizar parcialmente el usuario 42
DELETE /usuarios/42       → Eliminar el usuario 42

Fíjate en que los nombres de los recursos van en plural y se anidan de forma lógica: /usuarios/42/posts para obtener los posts del usuario 42.

Códigos de respuesta HTTP

El servidor responde con un código numérico que indica el resultado de la operación:

  • 200 OK: La petición fue exitosa.
  • 201 Created: El recurso fue creado exitosamente.
  • 204 No Content: La operación fue exitosa pero no hay contenido que devolver (común en DELETE).
  • 400 Bad Request: La petición es inválida (datos faltantes o mal formados).
  • 401 Unauthorized: No estás autenticado.
  • 403 Forbidden: No tienes permiso para acceder al recurso.
  • 404 Not Found: El recurso no existe.
  • 500 Internal Server Error: Error del servidor.

Formato de datos: JSON

La mayoría de las APIs REST modernas usan JSON (JavaScript Object Notation) para intercambiar datos. Es un formato ligero, legible por humanos y fácil de procesar por cualquier lenguaje. Un ejemplo de respuesta JSON:

{
  "id": 42,
  "nombre": "María García",
  "email": "maria@ejemplo.com",
  "activo": true,
  "fecha_registro": "2026-01-15"
}

Cómo consumir una API REST con JavaScript

En el navegador, la forma moderna de hacer peticiones HTTP es con la API fetch. Aquí tienes un ejemplo completo:

Petición GET

fetch('https://jsonplaceholder.typicode.com/users/1')
  .then(response => {
    if (!response.ok) {
      throw new Error(`Error HTTP: ${response.status}`);
    }
    return response.json();
  })
  .then(data => {
    console.log('Usuario:', data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Petición POST

const nuevoUsuario = {
  nombre: 'Carlos López',
  email: 'carlos@ejemplo.com'
};

fetch('https://jsonplaceholder.typicode.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(nuevoUsuario)
})
  .then(response => response.json())
  .then(data => {
    console.log('Usuario creado:', data);
  });

Usando async/await

La forma más legible y moderna de trabajar con fetch es usando async/await:

async function obtenerUsuario(id) {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);
    if (!response.ok) throw new Error(`Error ${response.status}`);
    const usuario = await response.json();
    return usuario;
  } catch (error) {
    console.error('Error:', error);
  }
}

obtenerUsuario(1).then(usuario => {
  console.log(usuario.nombre); // "Leanne Graham"
});

Cómo consumir una API REST con Python

En Python, la librería estándar urllib puede hacer peticiones HTTP, pero la forma más cómoda es con la librería requests. Si no la tienes instalada:

pip install requests

Petición GET con requests

import requests

response = requests.get('https://jsonplaceholder.typicode.com/users/1')
if response.status_code == 200:
    usuario = response.json()
    print(f"Nombre: {usuario['nombre']}")
else:
    print(f"Error: {response.status_code}")

Petición POST con requests

import requests

nuevo_usuario = {
    'nombre': 'Ana Martínez',
    'email': 'ana@ejemplo.com'
}

response = requests.post(
    'https://jsonplaceholder.typicode.com/users',
    json=nuevo_usuario
)

if response.status_code == 201:
    print('Usuario creado:', response.json())
else:
    print(f"Error: {response.status_code}")

API públicas para practicar

Si quieres practicar el consumo de APIs, aquí tienes algunas gratuitas y sin autenticación:

  • JSONPlaceholder: jsonplaceholder.typicode.com — API falsa para hacer pruebas con posts, usuarios, etc.
  • PokéAPI: pokeapi.co — Información completa de todos los Pokémon.
  • OpenWeatherMap: api.openweathermap.org — Clima y pronósticos (requiere API key gratuita).
  • GitHub API: api.github.com — Información de repositorios, usuarios y más.

Conclusión

Las APIs REST son el estándar de comunicación en el desarrollo web moderno. Entender cómo funcionan los métodos HTTP, los códigos de respuesta y cómo consumir APIs desde JavaScript y Python te abre las puertas a construir aplicaciones que se integran con servicios externos. La práctica es clave: elige una API pública, haz algunas peticiones y experimenta. Cuanto más las uses, más naturales te resultarán.
